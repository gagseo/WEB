Cache-Control 헤더를 통해서 캐시 정책을 지정할 수 있다.
		*ETag : 캐시에 저장된 컨텐츠와 서버의 컨텐츠가 다른지 검사하는 태그
			서버의 컨텐츠가 변경되면 ETag값이 바뀌게 된다.
		Cache-Control
			no-store : 캐시를 저장하지 않음
			no-cache : 매번 서버에서 ETag를 확인 하겠다.
					ETag가 변경되었으면 서버에서 자원을 다운 받고
					변경되지 않았으면 캐시에 저장된 자원을 사용하겠다.
			max-age : 캐시 수명, max-age에 지정한 시간이 지나지 않았으면
					서버에 묻지 않고 캐시에 저장된 자원을 사용
					지정한 시간이 지나면 ETag를 비교
					max-age = 0 == no-cache
